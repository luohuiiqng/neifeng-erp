# 内丰 ERP · HTML 交互原型说明

## 1. 文档目的

本文档说明基于 **Vue 3** 实现的设备厂商 ERP **可点击原型**：用于与客户或内部评审流程、页面结构与权限思路。**非生产系统**，无真实后端与持久业务数据（订单等为前端模拟数据；角色配置存浏览器本地）。

- **给业务同事（非技术）**：软件在做什么、怎么点、流程故事线 → 请先看 [`软件设计与交互说明（业务版）.md`](./软件设计与交互说明（业务版）.md)。  
- **给产品 / 开发（偏技术）**：交互步骤与路由级流程 → [`交互流程说明.md`](./交互流程说明.md)。

## 4. 本期范围（与前期共识一致）

**包含（原型级）：**

- 工作台概览、快捷入口  
- **生产订单**列表与详情（主信息、明细、并行事项、出货与财务轻量信息；业务上由总经理/授权岗位**直接下达**，非「销售订单」模块）  
- 采购申请列表/详情、采购订单详情（含**轻量到货登记**，不做库存账）；申请关联 **productionOrderId**  
- 出货确认表单（演示），关联生产订单  
- 财务应收列表（演示）  
- 审批中心列表与审批处理（演示）  
- **角色管理**：维护角色名称、编码、说明与菜单权限；**顶部「模拟角色」**切换当前身份以体验菜单与路由拦截  

**明确不包含（占位或下期）：**

- **销售模块**（客户、报价、合同、销售订单等）——侧栏 **「销售」** 为「下期」说明页  
- **车间执行**（工单、报工、工艺等）——侧栏 **「车间执行」** 为「下期」说明页（原「生产」执行深度）  
- **仓库**（库存账务、库位批次等）——同上  

**本地角色数据**：升级原型时若 `nf_erp_proto_roles_schema` 版本变化，会**清空自定义角色**一次并写入新默认角色（见 `src/stores/roles.js`）。

## 6. 角色与权限（原型逻辑）

### 6.1 默认角色

内置角色包括：**系统管理员、总经理、厂长、采购、财务、车间主任** 等；管理员拥有**全部功能**（`*`），其余角色按模块组合授权（见 `src/stores/roles.js` 初始数据）。**总经理默认不含「采购」权限**（`purchase`），与工作台快捷入口、详情「去采购申请」按钮一致隐藏。**本期无「销售」角色**；采购、财务可按需配置 **生产订单** 跳转权限。

### 6.2 权限项与菜单对应关系

权限 key 与侧栏菜单对应关系定义于 `src/mock/data.js` 的 `MENU_DEFS`（如 `production_order` → 生产订单，`roles` → 角色管理）。  

- 无权限的菜单**不显示**。  
- 直接访问无权限路由时，会**重定向到工作台**。  

### 6.3 操作权限（按钮级）

`src/mock/data.js` 中的 **`ACTION_DEFS`** 定义按钮权限 key（如 `action_mo_create` 新建生产订单、`action_mo_issue` 订单下发、`action_approve_close` 审批结案类等），与 `useRolesStore().can(key)` 配合。  

- **默认分工示例**：总经理可**新建**、**订单下发**、结案审批，**不**操作齐套转待出货与出货确认提交，不可进采购菜单；**厂长**决定出货（`action_mo_to_ship`、`action_ship_submit`），可批采购申请；财务可申请结案、不可点「同意结案」；车间主任仅看已下发订单；采购可提交采购申请；**生产部**维护并行事项。`canViewDraftProductionOrder()` 控制草稿可见性。  
- 在 **角色管理** 中与菜单一并勾选；`roles` schema 版本升级时会重置内置角色。  

### 6.4 模拟当前身份

顶栏 **「模拟角色」** 下拉框会切换 `currentRoleCode` 并写入 `localStorage`（键名见 `src/stores/roles.js`）。用于演示「同一套页面，不同角色看到不同菜单」。

### 6.5 角色数据持久化

在 **角色管理** 中的新增、编辑、删除及「恢复默认」会读写浏览器 **localStorage**。清除站点数据或换浏览器会恢复默认角色（除非重新配置）。

## 7. 推荐演示路径（给客户讲 5～8 分钟）

1. **工作台** → 看待审批与进行中生产订单。  
2. **生产订单** → 打开 `MO-2026-0319-001`，切换 Tab 看明细与**并行事项**（对应手绘流程三条线）。  
3. **采购** → 打开演示申请 `PR-2026-0320-01`，再进入采购订单 `PO-2026-0321-01` 看**到货登记**。  
4. **出货** → 演示提交（仅提示，无后端）。  
5. **财务应收** → 看未收/已收演示数据；可对已全款订单点「申请结案」跳转审批。  
6. **审批中心** → 处理待审批单（演示）。  
7. **角色管理** → 改某一角色权限；顶栏切换「模拟角色」验证菜单变化。  
8. **销售 / 车间执行 / 仓库** → 说明为下期模块。

## 8. 演示数据说明

- **生产订单运行时列表**：Pinia `useProductionOrderStore`，种子来自 **`src/mock/data.js`** 的 `productionOrders`，可持久化到浏览器 **localStorage**（见 `src/stores/productionOrders.js`）。  
- 采购单、审批等待办等仍为 **`src/mock/data.js`** 静态演示数据。  
- 与线下「生产单」类似的字段（**单位名称、机型、冷/热、缝包、台数、打孔、备注**）体现在订单明细行中。
